The basics of HTML

HTML stands for Hyper Text Markup Language.

HTML documents are composed of HTML elements. Each HTML element describes a type of document content.

Here's a Heading Level 1 element:

<h1>Welcome to our webpage!</h1>

Note that this element has 2 tags: an opening tag (<h1>) and a closing tag (</h1>). The tags themselves indicate the element type; between the tags is the content of the element

HTML elements can also be nested - that is, an element may contain one or more other elements inside of it!

Some HTML elements, such as the <br> line break element, do not have a closing tag.
These elements are called empty elements, because they cannot contain any content, including other nested elements.

Elements may also have attributes indicating additional properties/information about the element, which are specified within the opening tag.
For example, an <img> image element (see details below) may include attributes like src, alt, and title.

Fundamental Document Structure

Here is a simple but complete page.


    <html>
        <head>
            <title>Our Webpage</title>
        </head>
        <body>
            <h1>Welcome to our webpage!</h1>
            <p>There isn’t much here yet.</p>
            <p>But, there will be more very soon!</p>
        </body>
    </html>
  • HTML elements in a document are organized in a hierarchy, a lot like a family tree. The “root” of the tree is the html element.
  • Nested elements are called “children” of the element that contains them. The containing element is called the “parent” element. For example, the body element has 3 children: h1, and two p elements, to which the body element is their parent.
  • The HTML element always has exactly two children - the head and the body elements. These are sectioning elements with special purposes:
    1. The body element contains all of the elements that represent content to be displayed onscreen
    2. The head elements contains non-displayed information about the page, such as the tab title or other meta information about the site (for example, short descriptions and keywords used by search engines and social media)
  • Note the use of indentation to organize the nested elements. See the "Formatting HTML" section for tips on keeping things organized.

HTML elements

HTML Text elements

This is a non-exhaustive list of common HTML elements used to organize and structure blocks of text on a page. For a more in-depth tutorial on these common elements, check out MDN's HTML text fundamentals

  • The <p> element represents a paragraph.
  • The <h1>, <h2>,... <h6> elements represent six levels of section headings;<h1> is the highest level and largest text, <h6> is the highest level and largest text.
  • The <ul> and <ol> create Unordered (bulletted) lists and Ordered (numbered) lists, respectively. Both kinds of lists have <li> child elements (“list item”), which separate the different items of the list.
  • The <pre> ("preformatted") element renders its contents as written in the HTML file, including whitespace and linebreaks, usually in monospace font. This is helpful for displaying certain text like code or ASCII art.
  • The <br> element produces a line break; it is an empty element and thus has no closing tag or content.
    This is useful for poetry, lyrics, or other text content where the line breaks are specific, and should not be determined purely by the width of the text area.
    Generally, do not use <br> to do paragraph breaks or separate blocks of text; rather, separate the text into multiple elements.
  • There are many HTML elements that allow for in-line text semantics. Here are some common ones:
    • <strong>, which indicates important text and is typically rendered as bold.
    • <em>, which indicates emphasized text and is typically rendered as italic.
    • The <b> and <i> elements also do bold and italics. However, it is generally considered better practice to use <strong> and <em>.
    • The <u> element makes underlining. However, underlined text on the web usually implies links, so it's usually not a good idea to underline things.
    • The <code> element indicates code snippets, and usually renders in monospace font. It is often nested inside a <pre> ("preformatted") element.

For a few more advanced HTML text formatting elements, check out: MDN's Advanced Text Formatting

A full list of HTML text elements can be found here: MDN's HTML text elements' documentation

Other HTML elements

Here are some other kinds of (non-text) HTML elements.
  • The special <!DOCTYPE> element always appears as the first line of an html document, and declares the document’s type/version of HTML. Modern pages written in HTML5 need only have: <!DOCTYPE html>
  • The <title> element species the title displayed in the Tab bar at the top of the browser window. This goes in the <head> section of the HTML document.
  • <meta> elements also go in the <head> section of the HTML document. These contain “meta” information about the page, but that isn’t itself displayed ON the page. These may be used by the browser, search engines, social media platforms, etc.
  • The <img> element is an empty element which displays an image. The location of the image file must be provided as the src attribute. Optional attributes include alt for screen readers, and title for tooltip (mouse-over) text.
    Here's an example of an image element:
    <img src="picture.jpg" alt="A dog wearing sunglasses" title="Isn’t he cute?!?">
  • <a>, known as the anchor element, creates a hyperlink to a URL given in the href attribute.

Formatting HTML

  • When writing HTML, extra spaces, indentations, and linebreaks are not rendered by the browser. This might be frustrating at first, but it actually allows us to better organize the code without affecting the layout negatively. If you have a lot of content in a single element, you can spread it over several lines for viewability.
  • When nesting elements, use indentation to organize levels; each level of nesting increases by one indent. (A common exception are the head and body elements, since they are standard in every HTML document)
  • If we have parent elements stretched across multiple lines, we sometimes try to keep the opening and closing tags aligned at the same indentation.
